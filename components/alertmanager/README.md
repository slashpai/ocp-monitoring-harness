## Role in the Stack

Alertmanager handles alerts generated by Prometheus. It:

- **Deduplicates** alerts across Prometheus replicas
- **Groups** related alerts to reduce notification noise
- **Routes** alerts to the appropriate receivers based on label matchers
- **Silences** alerts during maintenance windows
- **Inhibits** lower-severity alerts when higher-severity ones are firing
- **Notifies** external systems (PagerDuty, Slack, email, webhooks, etc.)

## Deployment Topology

Two replicas form an Alertmanager cluster using a gossip protocol (mesh networking). Both replicas receive alerts from both Prometheus replicas. The cluster deduplicates so that each notification is sent only once.

Containers in each pod:

- `alertmanager` — Main Alertmanager process
- `config-reloader` — Watches for config changes and triggers reloads (auto-injected by Prometheus Operator)
- `kube-rbac-proxy-web` — AuthN/AuthZ proxy for web UI access (port 9095)
- `kube-rbac-proxy` — AuthN/AuthZ proxy for tenancy endpoint, fronts prom-label-proxy (port 9092)
- `kube-rbac-proxy-metric` — AuthN/AuthZ proxy for the metrics endpoint (port 9097)
- `prom-label-proxy` — Injects namespace label for tenant isolation (port 9096)

## Key Configuration

In `cluster-monitoring-config` ConfigMap under the `alertmanagerMain` key:

| Setting | Default | Description |
|---|---|---|
| `resources` | (defaults) | CPU/memory requests and limits |
| `volumeClaimTemplate` | (none) | PVC template for persistent storage |
| `nodeSelector` | (none) | Node scheduling constraints |
| `tolerations` | (none) | Tolerations for tainted nodes |
| `secrets` | (none) | Additional secrets to mount (e.g., for receiver auth) |

### Alertmanager Routing Configuration

Routing is configured via the `alertmanager-main` Secret in `openshift-monitoring`. The secret contains `alertmanager.yaml` with:

```yaml
route:
  receiver: default
  group_by: ['namespace', 'alertname']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 12h
  routes:
    - match:
        severity: critical
      receiver: pagerduty-critical

receivers:
  - name: default
  - name: pagerduty-critical
    pagerduty_configs:
      - service_key: <key>
```
